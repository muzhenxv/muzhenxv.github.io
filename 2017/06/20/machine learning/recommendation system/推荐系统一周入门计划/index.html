推荐系统一周入门计划

说明

计划从2017/06/17-2017/06/23利用一周时间实现推荐系统入门。

2017/06/17

看了项亮《推荐系统实战》前三章。

学习了SVD算法原理和相关python代码。主要是numpy.linalg.svd。参考[机器学习笔记]奇异值分解SVD简介及其在推荐系统中的简单应用。

寻找了Santander Product Recommendation作为实验数据。

学习了FP-growth算法寻找频繁项集。频繁项集挖掘算法之FPGrowth。

回顾了apriori算法。Apriori算法详解之【一、相关概念和核心步骤】。主要是支持度和置信度的概念需要掌握。

Santander Product Recommendation

该方法提供了一个非常好的特征思路。下个月买什么不仅跟历史偏好有关。也跟宏观的产品变化有关。计算出每个月各产品的总销售情况，并从中探索销售趋势，乃至于宏观角度的关联规则都是很有意义的。

Santander Product Recommendation

给出了一些标准化思路。比如年龄以100为最大值进行标准化。利用年龄信息来构造婚姻信息，根据西班牙的平均结婚年龄（分男女）来制造婚姻特征。

对产品进行了分析，直接剔除了不活跃的产品的候选可能。不活跃有两种可能：没人买，或者大家都稳定买以至于并不存在是新增产品的可能。同时探索了季节性产品的存在。季节性产品是非常重要的特征！

2017/06/20

粗略浏览了剩下的高投票discussion。除了有一个使用了svd。其他基本都是特征工程加上模型。并没有发现使用协同过滤算相似度的方法进行推荐的。

推荐系统暂时告一段落。能够理解并运用SVD，协同过滤，apriori和fp-growth我觉得暂时已经足够。
